## Project Overview

**PlatAlgo Relay** is an Electron + React desktop app that bridges MetaTrader 5 (MT5) with a cloud-based trading automation service. The desktop app itself has no local MT5 or Flask process — it communicates exclusively with the cloud bridge at `https://app.platalgo.com`.

## Commands

All commands run from `relay-ui/`:

```bash
npm run dev                 # Vite dev server only (port 5173)
npm run electron:dev        # Dev mode: Vite + Electron with live reload
npm run build               # TypeScript compile + Vite bundle → dist/
npm run electron:build      # Package desktop app (macOS + Windows)
npm run electron:build:mac  # macOS DMG + ZIP (arm64, x64)
npm run electron:build:win  # Windows NSIS EXE (x64)
npm run lint                # ESLint (TypeScript + React Hooks + React Refresh)
```

There is no test suite.

## Architecture

### Process Model

```
Electron Main (electron/main.cjs)
  └─ Preload Bridge (electron/preload.cjs)  ← IPC boundary; context isolation ON
       └─ Renderer: React app (src/)
```

The preload script is the only communication channel between Electron and React. It exposes a safe `window.bridge` API — never use Node APIs directly in renderer code.

### Renderer Structure

- **`src/store/appStore.ts`** — Single Zustand store for all app state: auth, relay dots (`bridge`/`mt5`/`broker`), `vpsActive`, logs, active tab, theme
- **`src/lib/api.ts`** — All cloud bridge API calls; imports `BRIDGE_URL` from `constants.ts`
- **`src/lib/bridge.ts`** — Typed wrapper around `window.bridge` IPC
- **`src/hooks/useRelay.ts`** — Polls `/dashboard/summary/login` every 10s; updates relay status dots; detects managed relays by `relay_id` prefix `managed-*`
- **`src/App.tsx`** — Restores auth from IPC on mount, starts polling

### Key IPC Methods (via `src/lib/bridge.ts`)

| Method | Purpose |
|--------|---------|
| `openExternal(url)` | Open URL in system browser — always use this, never `window.open` |
| `keyringGet/Set()` | OS native password storage (Keychain / Windows Credential Manager) |
| `lastUserGet/lastUserSet()` | Persist session across launches (writes to Electron user data dir) |
| `checkForUpdates()` / `onUpdateStatus()` | Auto-update via electron-updater + Cloudflare R2 |

### API Authentication

All requests to the cloud bridge use:
```http
X-User-ID: <user_id>
X-API-Key: <api_key>
```

### Component Layout

- `src/components/layout/` — `AppShell`, `Header`, `Sidebar`, `StatusPill`
- `src/components/connect/` — `SignInCard`, `MT5LoginCard`, `VPSCard`
- `src/components/dashboard/` — `DashboardPanel`
- `src/components/telegram/` — Telegram channel management + signal parsing
- `src/components/ui/` — Shared primitives: `Card`, `GoldButton`, `OutlineButton`, `Input`, `StatusRing`, `ScrollReveal`

## Key Conventions

- **Never hardcode the backend URL** — always import `BRIDGE_URL` from `src/lib/constants.ts`
- **Never use `window.open`** — use `bridge.openExternal(url)` so the system browser opens in Electron context
- **Theme** — toggled via `document.documentElement.dataset.theme` (`"dark"` | `"light"`); CSS variables respond to this attribute
- **Relay dot states** — `'online' | 'offline' | 'unknown'`; managed relay detection uses `relay_id.startsWith('managed-')`
- **Vite base path** — set to `'./'` for file:// URL compatibility in Electron production builds
